# UI 動作確認チェックリスト (Phase 2 + Phase 3)

Phase 2 / Phase 3 のリファクタリングはすべて「挙動・見た目を変えない構造変更」が原則です
(例外は明記)。ユニットテスト + E2E は green ですが、見た目のリグレッションと
実環境での挙動はテストでカバーしきれないため、以下を実機で確認してください。

凡例: ⭐ = 特にリグレッションリスクが高い箇所 / 🆕 = 意図的な挙動変更

---

## 1. Agent 詳細モーダル (PR-C: AgentDetailModal 1004→358行に分解) ⭐

最も大きく分解した箇所。見た目・挙動が完全に一致しているか重点的に確認。

- [ ] Settings → Agents から **既存 Agent の「編集」** を開く → モーダルが開き、name /
      description / アイコン / 色 / 公開・既定チェック / System Prompt / クイックアクション /
      公開先 / Skills / Tools が**従来どおり**表示・編集できる
- [ ] **「+ Custom Agent を追加」**(create モード)→ 雛形プルダウンが出て、選ぶと
      フォームがリロードされる
- [ ] アイコン/色のピッカーが正しく選択ハイライトされる(IconChoice / ColorChoice)
- [ ] **クイックアクション**: 1 行 1 件入力、空行無視、6 個以上 / 200 文字超 / 1024 byte 超で
      警告が出る(QuickActionsSection)
- [ ] **公開先 (AccessPicker)**: ユーザー/グループ/組織の検索・追加が動く
- [ ] Skills / Tools のチェックボックス ON/OFF が draft に反映される
- [ ] built-in Agent 編集時に **「初期値に戻す」** が出荷時 spec に戻す
- [ ] ⭐ **削除確認ダイアログ**(ConfirmDialog に汎用化): custom Agent の「削除」→
      「『○○』を…アーカイブします」のダイアログが出て、「削除する」で削除、
      「キャンセル」で閉じる。**`<p>`→`<div>` に変えたため上端の余白が数 px 異なる可能性**
      → 見た目に違和感がないか
- [ ] **エージェントデザイナーの提案から開く**(create-from-proposal): 「エージェント
      デザイナーによる提案」バッジ + 設計理由 details + 「自分で雛形から作り直す」リンクが動く
- [ ] 保存 / キャンセル / 閉じる(×)ボタンが従来どおり

> 🆕→見送り: design の AC-4(モード切替のみでの不要な再フェッチ抑制)は**本リファクタでは
> 適用していません**(挙動を完全維持する判断)。モーダルを開くたびに Agent 詳細を fetch する
> 既存挙動のままです。気になる体感の遅さがあれば別途対応します。

## 2. カスタム skill 追加モーダル (PR-D: SkillAddModal 637→236行に分解) ⭐

- [ ] Settings → 🧠 スキル → 「追加」→ モーダルが開く
- [ ] **「📤 ファイル」タブ**: `.md`(frontmatter 付き)をドロップ / 選択 → name・description が
      自動抽出され、プレビューが出る
- [ ] **`.zip` / `.skill`** をドロップ → 複数ファイルが展開され「N ファイル」と表示、
      ファイル一覧が出る
- [ ] サイズ上限超過(8 MB 超)/ 非対応拡張子 / frontmatter に name 無し → **エラー文言が出る**
- [ ] アップロード後に name / description を編集できる、「差し替え」で選び直せる
- [ ] **「📝 直接入力」タブ**: name / description / SKILL.md 本文を直接入力できる
- [ ] **編集モード**(既存 skill を編集): name が読み取り専用、本文編集で「保存 (新バージョン)」
- [ ] アップロード / 保存が成功し、Anthropic に同期される

## 確認方法のヒント

- ローカルで `pnpm plugin:build` → kintone にアップロードして実機確認、または
  `/run` / `/verify` skill で起動して確認。
- 見た目のリグレッションは、可能なら Phase 2 着手前のスクリーンショットと比較。
- 問題が見つかったら、どの画面・どの操作・期待と実際を教えてください。該当 PR を特定して直します。
