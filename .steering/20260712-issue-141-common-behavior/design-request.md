# Claude Design 依頼ブリーフ — 共通 base システムプロンプト編集 UI（#141 / Plugin Config）

Cowork Agent for kintone の **Plugin Config 画面（admin 専用の設定画面）** に、**全エージェント共通の「base システムプロンプト」を編集・リセットできるセクション** を設計依頼する。成果物は既存ハンドオフ（`docs/design-handoff/memory-settings/` 等）と同じ流儀の **hifi 仕様 + 参照 TSX + prototype** を `docs/design-handoff/base-prompt-config/` として作ること。

**既存 ConfigScreen の意匠・パターンを最大限流用**してほしい（新規発明は最小に）。

---

## 0. 大前提（プロダクトモデル）

- Cowork Agent の各エージェント（業務 / カスタマイザー / アプリデザイナー / エージェントデザイナー / admin 作成の Custom）は、システムプロンプトが **base（共通の作法）+ persona（エージェント固有）** の2層で構成される。
- **base** = 全エージェントに効く「基本作法」（トーン/書式・誠実さ・ツールの使い方・メモリの扱い・メタ振る舞い）。**tenant 全体で1つ**。
- この UI は admin が **base を編集**し、必要なら **デフォルトに戻す** ためのもの。**persona（各エージェント固有）はこの画面では扱わない**（別途エージェント編集で）。
- base はコードに**既定値**を持ち、Config で override（未設定なら既定を使用）。変更は**次の新規会話から反映**される（既存の会話や再デプロイは不要）。

**スコープ外**: persona 編集、エージェント個別設定、プロンプトのバージョン履歴/差分、プレビュー実行。

---

## 1. 置き場所 / 既存 ConfigScreen の構造（流用元）

ConfigScreen は **max-width 720px・admin 専用**の縦積みカード群:
- `Step 0. Cloudflare Workers をデプロイ (任意)` / `Step 1. Worker URL + API Key` / `Step 2. OAuth 案内` / `Step 3. kintone OAuth` / `MCP サーバー` セクション。
- 各セクションは `<section class="rounded-[12px] border border-card-border bg-card p-[16px]">` + `<h2 14px/semibold>` + 補足 `<p 11px/muted>` + `<label 12px>` + input（`rounded-[8px] border border-card-border bg-bg px-3 py-2`）。
- 画面下部に「保存」ボタン（既存）。

**本セクションはこの語彙で、Step 群の後・保存ボタンの前あたりに 1 枚追加**する想定（採寸・正確な位置は Design 判断）。**上級者向け設定**なので、既定は畳んでおく（アコーディオン）とノイズが少ない、という方針も歓迎。

## 2. 設計してほしいサーフェス — 「共通システムプロンプト (base)」セクション

### 要素
- **見出し**: 例「共通システムプロンプト（全エージェント共通の作法）」+ 「上級者向け」バッジ or 補足。
- **補足文**: 「全エージェントに共通で効く基本作法です。各エージェント固有の指示（persona）は含みません。変更は次の新規会話から反映されます。」
- **テキストエリア**: 複数行・等幅・リサイズ可・**文字数カウンタ**（上限目安あり、例 20,000 字）。
- **状態表示**: 現在「**既定を使用中**」か「**カスタム（override）保存済み**」かが一目で分かるチップ/ラベル。
- **「デフォルトに戻す」**: override をクリアして既定へ戻すボタン（実行前に確認）。override が無い（既定使用中）ときは無効 or 非表示。
- **既定の扱い**: 未設定時、テキストエリアは空でプレースホルダに「未設定のため既定の作法を使用します」。任意で「既定を読み込む/差し込む」導線（既定テキストをエディタに流し込んで編集の起点にできる）。
- **保存**: 既存の画面下「保存」に含める（このセクション専用の保存ボタンは作らない）か、セクション内で完結させるか → Design 提案可（既存は画面一括保存なので合わせるのが自然）。

### 状態バリエーション（各モックが欲しい）
1. **既定使用中 / 未編集**（テキストエリア空 + プレースホルダ + 「既定を使用中」チップ + リセット無効）。
2. **編集中（override 入力）**（テキスト入り + 文字数 + 「カスタム」チップ）。
3. **override 保存済み**（保存後の落ち着いた状態 + 「デフォルトに戻す」有効）。
4. **リセット確認**（ConfirmDialog: 「共通プロンプトを既定に戻します」）。
5. **（任意）文字数超過 / バリデーション警告**。
6. **（任意）アコーディオン折りたたみ時**の見え方。

### インタラクション
- 「デフォルトに戻す」→ 確認（既存 `ConfirmDialog` 流儀）→ テキストエリアが空（既定使用）に。
- 「既定を読み込む」（任意）→ 既定テキストをエディタに差し込む（override 起点）。
- 変更は下書き扱い、**画面下の「保存」で確定**（既存挙動に合わせる場合）。未保存の変更がある注記があると親切。

## 3. 流用してほしい既存パターン / 意匠

- **ConfigScreen のセクションカード**（`bg-card` / `border-card-border` / h2 / label / input）＝ 第一参照。
- **CA ブランド**: teal アクセント（`--cw-accent`）+ 既存 `--cw-*` トークン。
- **ConfirmDialog**（リセット確認）/ **Banner**（未保存/バリデーション）＝ 既存を再利用。
- 「上級者向け」バッジや折りたたみは既存にあれば流用、無ければ最小の inline SVG chevron。

## 4. 制約 / 注意
- デスクトップ幅のみ（ConfigScreen は kintone admin 画面）。ダーク/ライトは既存準拠（ライト主体）。
- アイコンは inline SVG（外部フォント不可）。文言は日本語。
- **admin 専用・上級者向け**。多くの admin は触らないので、既定はノイズにならない見せ方（畳む/薄める）を優先。
- 破壊的ではないが影響が広い（全エージェント）ため、リセットは確認を挟む。

## 5. 成果物
`docs/design-handoff/base-prompt-config/` に、既存ハンドオフと同じ構成で:
- **hifi 仕様**（レイアウト・状態・トークン・採寸）
- **参照 TSX**（`BasePromptSection.tsx` 等。ConfigScreen の 1 セクションとして差し込める形。props 例は下記）
- **prototype**（状態バリエーションを切り替えられる素の jsx/html）

既存 ConfigScreen の 1 セクションとして自然に馴染ませることを最優先。

## 参考: データ形状（実装側の確定仕様）

```ts
type BasePromptSectionProps = {
  /** 現在の override（未設定 = 空文字。空なら既定を使用）。 */
  value: string;
  onChange: (v: string) => void;
  /** コードの既定 base（「既定を読み込む」やプレースホルダ表示に使う）。 */
  defaultBase: string;
  /** override をクリアして既定へ戻す（確認後）。 */
  onResetToDefault: () => void;
  /** 文字数上限（目安）。 */
  maxLength?: number;
};
// 保存は ConfigScreen 全体の「保存」に集約（このセクション単体の保存ボタンは持たない想定）。
// 「既定使用中 / カスタム」は value の空判定で表示。
```
